PART 2: DESIGN (Answer ALL questions)

1. What overall style will best represent you (minimalist, creative, professional, etc.)? Creative and laid back. Nothing too fancy but not too bland or "professional" looking since it's not the sort of vibe I wanted to portray.

2. What color scheme will you use and why? Primarily dark colors with blue. I just really prefer dark mode on everything (it's one of the best modern features in tech) since it is better on my eyes and I'm sure the same goes for most people. Blue is my favorite color so it would be fitting to have a website that represents me to have that color

3. What fonts will you use for headings and body text? I have taken a liking to a certain font called "FOT-Yuruka" so it will be 90% of the fonts for headings and body texts.

4. How will your design reflect your personality or field? A lot of what I make and want to make involves doing something unique. I am the type of person to dump whatever I find cool and interesting into one giant place.

5. What layout will your homepage follow? For now it will basically be a collection

6. How will you organize project sections visually? Chronological order just to see my own progress and change in style. I will also organize it by type (Websites then Apps and so on...)

7. Will the site be mobile-friendly? How will you ensure responsiveness? I would like to make it mobile friendly with the help of some tools like Bootstrap and CSS manipulation.

8. What visual hierarchy will guide visitors? BIG HEADING then small text as well as color contrast. I will make sure to use a lot of headings for each section and pictures to go along with it

9. How will consistency be maintained across pages? I've thought of using different themes for each page but for now a consistent color scheme, font, and asset placement will be similar throughout the pages.

10. How will accessibility be considered (contrast, font size, readability)? For people that REALLY need it for some reason, there is a light mode toggle. Since FOT-Yuruka is a considerably large font, I hope it will also effectively solve any readability issues that may arise.

11. Will you use icons, images, or illustrations? Why? To better showcase my projects I shall use previews and images of those projects. I've also used many images from my favorite games, anime, and other medias to further personalize the design, hoping that visitors may have the same interests and appreciate it being there.

12. What portfolio websites inspired your design? Linktree, Carrd, some portfolio websites made by peers, content creators, vtubers.

PART 3: INTERACTIVITY (Answer ALL questions)

1. What interactive elements will your site include (navigation menus, buttons, forms)? Navigation bar inspired by Undertale/Deltarune, hoverable images, custom cursor, light/dark mode toggle, links to project repos, buttons, etc.

2. Will your site include a contact form? How will it work? The most I can do is leave my email and social tags for any contact info since my phone number is on my resume so that only employers would really get it more than normal visitors.

3. What JavaScript features will you implement? light/dark mode, back to top clickable when you scroll down and vanishes when you are at the top, image carousel if possible.

4. How does interactivity improve the user experience? I plan on making my site more than just a project and experience archive. I also want it to be as fun as I can possibly make it by using unnecessary but visually amusing elements without destroying the original vision I had for the website. So the way the cursor changes, gif behaviors, button animations, text animations, will all be implemented with more to come to make the user experience more enjoyable and will hopefully convey to the user that I really do these things for fun.
